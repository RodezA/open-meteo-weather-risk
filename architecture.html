Key Design Decisions

Engine as Pure Python

The risk engine has zero framework dependencies. It takes a HourlyWeather object and returns List[HourlyRisk] — runnable identically in Lambda, a Streamlit process, or a local test. No refactor needed to scale.

DynamoDB for Risk Cache

Risk scores are pre-computed on a 15-min schedule and cached by (lat, lon, hour). API reads hit DynamoDB — not Open-Meteo — giving sub-10ms reads at scale. Cache TTL matches Open-Meteo's hourly update cadence.

SQS for Ingest Decoupling

Weather fetches are queued, not inline. A single API request never waits on Open-Meteo. The DLQ captures failed location refreshes for replay without data loss. Backpressure is automatic.

SSM for OSHA Thresholds

Wind, precipitation, and visibility thresholds are stored in SSM Parameter Store, not hardcoded. Safety managers can update OSHA limits without a code deployment. Lambda reads at cold start and caches in memory.

S3 Parquet Archive

Every raw Open-Meteo response is archived as Parquet in S3. Enables retroactive analysis (e.g., "did STOP conditions correlate with reported incidents?"), regulatory audit trails, and ML training data for future models.

EventBridge Scheduled Refresh

All registered sites are refreshed every 15 minutes via EventBridge → SQS → Lambda. API reads are always cache hits. Alert latency is bounded at 15 minutes — acceptable for operational safety workflows.

Scalability & Trade-offs

Scaling to Thousands of Sites

  • SQS FIFO batches location refreshes — Lambda concurrency scales automatically
  • Open-Meteo rate limits handled via exponential backoff in the ingest Lambda
  • DynamoDB on-demand billing — no capacity planning at any volume
  • CloudFront caches API responses at edge for identical lat/lon queries

High-Frequency Updates

  • 15-min refresh is the practical floor (Open-Meteo updates hourly)
  • For sub-minute alerting: switch ingest trigger to IoT / on-site sensor feed
  • WebSocket endpoint via API Gateway for push alerts to site supervisors
  • DynamoDB Streams can trigger alerts without polling

What I'd Prioritize Next

  • Multi-source weather: NWS API as fallback if Open-Meteo is unavailable
  • Geocoding endpoint (Nominatim) so users search by address, not coordinates
  • Historical risk replay from S3 archive for incident correlation
  • Per-site threshold overrides (crane manufacturer limits vary)

Prototype Trade-offs (Consciously Made)

  • Single weather source — acceptable for POC, not production
  • Thresholds hardcoded in engine — SSM migration is a 1-day task
  • No auth layer — out of scope for this challenge
  • Streamlit replaces API Gateway for demo simplicity — engine is portable
